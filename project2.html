Cipher Application

What is a rail fence cipher?

A rail fence cipher is an ancient method of encryption in which a message would be written on a piece of fabric. This collection of words would look random until wrapped around a device called a scytale. Once assembled the recipient is about to decipher the message.

As Part of my object-oriented programming module, I was assigned a project that involved creating a java application with a simple user menu. The function of the application is to take in text and encrypt it using a rail fence cipher. The outline specified that the application should have a simple navigation menu and be able to handle files or a URL as the input. A two-part variable was to be input to determine the number of rails (or rows) and the row in which the text would begin in.

As seen in the menu below there are five options you can choose from. Once the details are added in the first option the encrypted message can be displayed. Then the cipher text received can be parsed into to decrypt option which returns the original message. The fourth menu option is to display the Rail fence pattern created within the application.

Implementation

To hit the requirements, I first designed a simple menu class that printed a list of choices to the console. Using the scanner, the program read user input and related that to decision making and the addition of variables. The variables the user chose would determine how many rows would appear in the pattern and what row it would begin. Once these were assigned the user was asked to parse a file path or URL.

I created a class to handle the files. The Url stream was declared and wrapped in an input stream reader and a buffered reader. Using a loop each line was added to a String variable to be returned for use in other classes. The file handling was similar but the file was wrapped in a file input stream, Input stream reader and a buffered reader.

The Rail Fence cipher class was made to handle the encryption, decryption, and creation of the matrix(pattern). I created a method that would create a blank matrix using the inputted number of rows and the number of characters in the message. Another method takes the plain text the user entered and parses it into the matrix by starting at the users chosen row and traversing through the rows and columns to create a zig zag. When the encryption method is invoked, it reads through the matrix each column in each row and outputs any character it finds to cipher text element. To decrypt this cipher text is added to each column of each row where a character is expected. To return the original text the program traverse diagonal through the matrix.
