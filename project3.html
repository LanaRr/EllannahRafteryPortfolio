Database Design

The image above is the Entity relationship diagram that shows the database i design for a dental practice. This diagram is denoted with crows feet notation. Crows feet notation is a simple way to show the relationships between the entitys.

To start off the design of the database we have the patients table. this stores all the contact information for the patient. Branching off that we have a patient history table which stores personal informaton about the patient past appointments and medical conditions and treatments. The appointments table holds date and time information. Some patients have specialist dentist so a table is created to hold infotmation and expertise.There is a table for bills that stores one or more bills for patients.This records cancellation fees and unpaid bills. This dental practice offers pay later option in which is stored in its own table.
